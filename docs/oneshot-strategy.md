# 원샷 전략 — 한국 시장에서 바운티 보드를 인게이지시키는 법

**Status:** strategy · **Owner:** founder · **Last updated:** 2026-07-19
**전제:** `docs/bounty-model.md`(비현금 크레딧 바운티 + holdout 자동검증)가 코어.
이 문서는 그 코어를 **한국 시장에서 사람들이 실제로 궁금해하고 돌아오게 만드는
포맷**으로 좁히는 전략이다.

## 1. 시장 리서치 요약 (2026 상반기)

| 발견 | 근거 | 함의 |
|---|---|---|
| **바이브 코딩이 국민적 열풍** | Collins '올해의 단어', "3일 만에 론칭" 보도에 24만 건 쇄도, 50대 비개발자까지 유입, 바이브코딩 공모전 개최 | 잠재 참여자가 개발자를 훨씬 넘어섬. "프롬프트로 만든다"는 개념을 설명할 필요가 없어짐 |
| **대회 문법이 이미 대중화** | 전국민 AI 경진대회 총상금 ~30억(과기부), 네이버 48시간 AI 해커톤 상금 3.5억, DACON 13만 팀 | "과제 → 마감 → 순위"에 참여하는 습관이 이미 있음. 상금 없이도 문법 자체는 통함 |
| **"원샷"이 이미 밈** | 인스타/릴스에서 "원샷 제작 — 말 한마디로 앱 완성" 포맷 유행, 커서·클로드 채널들 | 우리가 용어를 만들 필요가 없음. 유행어에 올라타면 됨 |
| **개발자 커뮤니티는 정반대로 회의적** | GeekNews의 Claude Code 논쟁은 대체로 비판적, 화두는 멀티에이전트·컨텍스트 엔지니어링·CLI 귀환 | "말 한마디로 앱"을 못 믿는 층이 콘텐츠의 반대 진영을 자처해줌 |

### 결정적 간극

> **대중은 "말 한마디로 앱"을 믿고 싶어하고, 개발자는 못 믿는다.
> 이 논쟁을 '실행'으로 정산하는 자리가 비어 있다.**

릴스는 체리픽이고 GeekNews는 말싸움이다. 둘 다 증거가 없다. poolproof의
holdout 엔진은 정확히 이 논쟁의 심판이 될 수 있는 유일한 구조다.

## 2. 포맷: 원샷 챌린지

바운티 보드 위에 얹는 **제약 하나**가 전체 전략이다 (Setlog 교훈: 제약이 곧
브랜드):

> **프롬프트 1개. 실행 1번. 수정 없음. 숨은 테스트가 판정.**

- 참가자는 코드가 아니라 **프롬프트 한 개**를 제출한다.
- 플랫폼이 같은 모델에 프롬프트를 넣고, 나온 코드를 **그대로** 실행한다.
  재시도·수정·이터레이션 없음.
- public 통과 → holdout 공개 → 살거나 죽거나. (기존 LiveVerdict 텐션 그대로.)

### 왜 원샷이 쐐기(wedge)인가

1. **참여 장벽이 프롬프트 한 줄.** 바이브코딩 열풍의 비개발자까지 전부
   잠재 참가자. "내 프롬프트 실력이 몇 점인가"는 판별게임의 "내 눈 몇 점인가"
   보다 훨씬 실용적인 자기증명 (이력서에 쓸 수 있는 능력).
2. **공정하고 이해 쉬움.** 같은 모델·같은 1회 실행 → 순수하게 프롬프트 실력
   비교. "누가 더 좋은 GPU/에이전트 셋업을 가졌나"가 아님.
3. **비용 통제.** 제출당 모델 호출 1회 + 실행 1회. 무한 이터레이션 없음.
4. **프롬프트가 곧 콘텐츠.** 성공한 원샷 프롬프트는 공개 박제 → 그 자체가
   학습 자료("이렇게 쓰니까 되더라"). "하루 만에 만들기" 유료 강의가 팔리는
   시장에서 이건 공짜 대체재 = 유입 자석.
5. **논쟁의 심판 포지션.** "원샷 성공률 — 과제 X에서 모델 Y는 N%"는 릴스도
   GeekNews도 못 내는 숫자. 양 진영 모두에게 인용당한다.

## 3. 인게이지 루프

```
릴스/커뮤니티에서 "이 프롬프트, 원샷에 될까?" 카드 목격
        │
        ▼
프롬프트 1줄 제출 (로그인만, 마찰 최소)  ←── 비개발자도 가능
        │
        ▼
LiveVerdict: public 쫙 → 숨은 테스트 한 칸씩 → 🟩 or 💀
        │
        ├─ 성공 → 크레딧 + 프롬프트 명예의 전당 박제 → 공유
        ├─ 실패 → "holdout #3에서 사망" 박제 → "다른 프롬프트로 재도전" (다음 과제)
        ▼
내 원샷 성공률 프로필 축적 → "나 원샷 상위 5%야" → 공유 → 신규 유입
```

- **긴장의 원천은 낮은 성공률.** 난이도 가드레일: 원샷 성공률 10–40% 밴드
  유지 (너무 쉬우면 스펙터클 없음, 0%면 좌절). holdout 설계로 조정.
- **실패도 콘텐츠** (Red Wall 원칙): "가장 아깝게 죽은 프롬프트" 주간 회고가
  성공 사례보다 더 공유된다.
- **크레딧 경제 접속:** 원샷 성공 = 크레딧 획득의 기본 경로. 모은 크레딧으로
  내 바운티(출제)를 건다 → earn-to-post 루프의 진입로가 원샷.

## 4. 주간 원샷 리그 (appointment viewing)

- 매주 같은 요일·시간에 새 과제 1개 공개. 마감까지 1인 1프롬프트.
- 마감 후 **일괄 실행 + 판정을 라이브로 공개** (48시간 해커톤 문법의 30분
  압축판). 결과 = 리그 순위 + 모델별 원샷 성공률 갱신.
- 시즌제 (10주 = 1시즌). 시즌 랭킹은 스크린샷 공유의 대상.

## 5. 과제(테마) 선정 기준

한 문장에 이해 + 한국적 공감 + killer holdout이 자연스러운 것:

- **이미 보유:** josa(조사), 맞춤법 교정기, 수능 국어, 로또 분배, slugify-korean
- **추가 후보:** 카톡 정산 나누기(더치페이 반올림 엣지), 연말정산 간이 계산,
  최저임금/주휴수당 계산기, 사업자번호 검증, 만 나이 계산(생일 엣지),
  전세 계약 체크리스트 파서, 명절 KTX 예매 시간표 파싱
- **원칙:** holdout이 "아 그 엣지케이스!" 하고 무릎 치게 되는 과제만.
  (서울+(으)로 급의 함정이 없는 과제는 원샷 100%가 나와서 스펙터클이 없음.)

## 6. 배포 채널 (순서대로)

1. **릴스/쇼츠 포맷 공략** — "이 프롬프트, 원샷에 될까?" 15초: 프롬프트 보여주고
   → LiveVerdict 판정 화면 → 💀 or 🟩. 기존 원샷 밈 포맷에 '진짜 판정'만 얹음.
2. **GeekNews/커리어리/OKKY 시딩** — 개발자용 앵글은 반대로: "바이브코딩
   과대광고를 숫자로 검증하는 보드". 회의론자가 우리 데이터를 무기로 쓰게 함.
3. **바이브코딩 공모전/해커톤 협업** — 이미 열리는 행사들의 '검증 인프라'로
   들어가기 (우리 holdout 엔진이 심판).
4. **프롬프트 명예의 전당 SEO** — 성공 프롬프트 아카이브가 "AI로 X 만들기
   프롬프트" 검색 유입을 먹음.

## 7. 지표

- **북극성: 주간 원샷 제출 수.**
- 보조: 원샷 성공률 밴드(10–40%) 유지 여부, 제출자 중 재제출(다음 주 복귀) ≥ 35%,
  판정 화면 공유율, 원샷→바운티 출제 전환율.
- 논쟁 지표: 모델별 성공률 표가 외부(커뮤니티/미디어)에 인용된 횟수.

## 8. 지금 코드베이스에서 만들 것 (build order)

1. **원샷 제출 흐름** — 프롬프트 입력 → `ai.ts`로 모델 호출 → 생성 코드를
   `runner.ts` 하드닝 실행 → 판정 기록. (엔진 전부 재사용, 새 조립만.)
2. **LiveVerdict 실전 연결** — 데모가 아니라 내 제출의 판정을 라이브로.
3. **원샷 성공률 보드** — 과제 × 모델 × 성공률. `verification_runs` 집계.
4. **주간 리그 스케줄러** — 주 1회 과제 공개/마감/일괄 판정 (`/api/cron` 재사용).
5. 크레딧 원장 (bounty-model.md의 2번) — 원샷 성공 보상과 연결.

## 9. 리스크

| 리스크 | 대응 |
|---|---|
| 모델 API 비용 폭증 | 1인 1프롬프트/과제 + 주간 리그 마감제로 호출량 상한. 크레딧으로 추가 제출 게이트 |
| 프롬프트 인젝션("테스트를 통과한 척해라") | 실행은 코드만, 하드닝 러너 유지. holdout은 생성 시점에 모델이 못 봄 — 구조적 방어 |
| holdout 유출 (성공 프롬프트 공개로 역추정) | 프롬프트는 공개하되 holdout 케이스는 영구 비공개 + 과제별 holdout 로테이션 |
| 원샷 성공률 0% 과제 (좌절) | 난이도 밴드 모니터 + public 테스트로 부분 점수 표기 |
