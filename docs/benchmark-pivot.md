# The un-gameable AI capability benchmark

**Status:** proposal · **Owner:** founder · **Last updated:** 2026-07-13
**전신:** poolproof (pay-on-green escrow) → 판별 게임 → **AI 실행 벤치마크.**
돈 레이어도, 사람-vs-AI 판별 게임도 접는다. 남기는 유일한 자산:
**holdout + 진짜 CI 실행으로 "조작 불가능한 판정"을 만드는 엔진.**

---

## 1. 한 줄 정의

> **"AI가 진짜 뭘 할 수 있나 — 학습으로 미리 못 본 숨은 테스트로만 채점하는,
> 조작 불가능한 실행 벤치마크."**

파는 것은 게임이 아니라 **정직한 능력 측정치.** "AI가 이거 된다/안 된다"는
지금 전부 vibes로 굴러간다 — 체리픽된 데모, 오염된 벤치마크, 트위터 언쟁.
우리는 AI가 **미리 볼 수 없는 테스트**로 진짜 실행해서, 됐으면 됐고 죽었으면
정확히 어디서 죽었는지 박제한다. 데모로 못 속인다.

---

## 2. 왜 "미국 벤치마크"의 다음 버전인가

SWE-bench · ARC-AGI · GPQA · LMArena — 미국에서 AI 능력이 신뢰성 있게 정산되는
방식. 그런데 이들 전부가 같은 치명상을 공유한다:

- **오염(contamination).** 테스트셋이 학습 데이터에 새어들어간다. "SWE-bench
  Verified" 점수가 시간이 갈수록 부풀고, 아무도 벤치마크 숫자를 못 믿게 됐다.
- **게이밍.** 랩이 자기 모델에 유리한 셋을 고르고, 프롬프트를 맞춘다.

poolproof의 holdout 메커니즘이 이 둘을 **구조적으로** 막는다:

1. **공개 테스트(public)** — 빌더/모델이 본다. 뻔한 케이스.
2. **숨은 테스트(holdout)** — 절대 공개하지 않는다. 점수를 실제로 가르는 함정
   케이스. 모델이 **학습으로 미리 볼 수 없으므로** 오염이 원천 불가능.
3. **재실행 불가.** 한 번의 실행이 기록. 여러 번 돌려 최고점만 취하는 게이밍 차단.

`specs/josa` 가 이 원리의 살아있는 증거다:
- public: `책→책은`, `나무→나무는` (받침 토글이면 다 통과)
- holdout: `서울+(으)로→서울로` (ㄹ받침 예외), `GitHub+이/가→GitHub가`
  (비한글 종결) — **진짜 이해했는지** 가르는 케이스.
- public 다 통과하고 holdout에서 죽으면 = **"되는 척했지만 실은 못 함."**

---

## 3. 헤드라인 지표 — 남들이 못 내는 숫자

일반 벤치마크는 "통과율 %" 하나. 우리는 그 위에 아무도 못 내는 지표가 있다:

> **홀드아웃 갭(holdout gap) = public 통과율 − holdout 통과율
> = "AI가 되는 척하지만 실제론 못 하는 정도."**

모든 모델은 보이는 테스트에서 잘한다. 진짜 능력은 **갭**에서 드러난다. 이건
과제마다 방화벽 친 holdout 셋을 우리만 갖고 있어서 가능한 측정치다.

---

## 4. 참여 & "거는 것" (돈 아님)

핵심은 **"AI가 진짜 뭘 할 수 있냐"** 를 사람들이 실행으로 정산하는 것. 참여는
두 방향, 거는 것은 **공개·영구 기록에 남는 자기 판단.**

### 4.1 과제 출제자 (frontier probing) — primary supply
- "AI는 이건 못 해" 싶은 진짜 어려운 과제 + **killer holdout** 을 제출.
- 프런티어 모델이 그 과제에서 죽으면 → 당신은 **진짜 능력 갭을 발견**한 것.
  당신 이름으로 귀속된다. (ARC-AGI·GPQA가 전문가 기여 문제로 가치를 얻는 구조와
  동일 — adversarial, expert-sourced.)
- 거는 것: **"AI는 여기까지다"라는 당신의 판단.** 실행으로 맞았는지 정산된다.

### 4.2 빌더 / 모델 제출자
- 자기 에이전트·모델을 실제 과제에 붙인다. 실행은 진짜, 결과는 영구 공개.
- green도 holdout-death도 이름 붙어 박제 — 데모가 아니라 검증된 전적.

**돈이 아니라, 틀리면 깎이는 "프런티어에 대한 판단력"을 건다.** holdout 숨김·
재실행 불가라서 그 판단이 진짜 의미를 갖는다.

## 5. 무엇을 버리나 (non-goals)

- **사람 vs AI 판별 게임 전체.** (`/play`, streak, 🟩🟥 그리드 공유, 리더보드
  랭크 — "유치한" 스펙터클 레이어 전부.)
- 돈/베팅/에스크로 **정산** 레이어. (엔진은 남기되 "돈"은 안 건다.)
- Wordle-share·콜 GREEN/RED 같은 관중 게임화.

## 6. 이미 있는 것 vs 만들 것

**있는 것 (재사용):**
- `runner.ts` + `specs/_harness.mjs` — 하드닝된 실제 실행 (clean env, 권한 모델,
  타임아웃). public+holdout 채점 → `recordRun`.
- `specs/*/{public,holdout}.test.mjs` — 실제 과제 5종 (josa, markdown-alerts,
  wordle-solver, slugify-korean, iso-duration).
- `verification_runs` / `test_results` 영속 — 실행 기록 데이터 모델.
- `grid.ts` — 실행 결과의 텍스트 압축 (공유 스펙터클 말고 **투명한 로그**로 재프레임).

**만들 것 (build order):**
1. **랜딩 리프레임** — 판별게임 히어로 → "조작 불가능한 AI 실행 벤치마크"
   + holdout 갭 설명 카드. (이 PR)
2. **벤치마크 보드** — 과제 × 모델 매트릭스: public/holdout 통과율, 홀드아웃 갭.
   실제 `test_results` 로 채운 정직한 표.
3. **라이브 모델 실행** — `ai.ts` 로 실제 모델(Claude/GPT/…)을 과제에 붙여
   제출물 자동 생성 → 실행 → 보드 갱신. (현재는 손으로 놓은 `.mjs` 제출물.)
4. **과제 출제 흐름** — killer holdout 포함 과제 제출 + 검토 게이트.

## 7. 측정이 모든 걸 결정한다

북극성: **보드에 실린, 신뢰할 수 있는 (모델 × 과제) 실행 셀 수.** 부차: 홀드아웃
갭이 큰 과제(= 진짜 프런티어를 건드린 과제) 발굴 수, 인용/공유 시 "오염 불가능"
주장이 discourse를 만드는가. 판별게임의 Daily Plays·공유율은 폐기한다.
