# 오늘의 판별 — Product Brief (PRD · ICP · Value Prop · Moat)

**Status:** living doc · **Owner:** founder · **Last updated:** 2026-07-13
**전신:** poolproof (pay-on-green escrow). 돈 레이어를 접고, 핵심 자산인
**"판정을 스펙터클로 만드는 엔진 + 우리가 정답지를 소유하는 구조"** 만 가지고
사람 vs AI 판별 게임으로 피벗. 트래픽 우선, 수익화는 뒤.

---

## 1. 한 줄 정의 (Core Value)

> **"AI가 사람 행세를 하는 시대에, 내 감각이 아직 통하는지 — 그리고 내 글이
> 아직 사람으로 읽히는지 — 매일 확인시켜 주는 것."**

파는 것은 게임도 탐지기도 아니라 **보정된 자신감(calibrated confidence)**.
사람들은 매일 "이거 AI가 쓴 건가?" 의심하지만 자기 판단이 맞는지 확인할 방법이
없다. 우리는 정답을 알고 출제하므로(정답지를 우리가 만들었으므로) 그 확인을
**정직하게** 해줄 수 있는 유일한 곳이다.

**제1원칙 (poolproof 홀드아웃에서 계승):** *정답을 아는 것만 판정한다.*
- 게임 = 우리가 사람/AI 쌍을 만들어 정답을 앎 → 채점 가능. ✅
- 실제 제3자 메시지 판정기 = 정답지 없음 → 우리가 GPTZero처럼 틀리는 탐지기가
  됨. 오판 하나(무고한 애인/직원을 AI로 낙인)로 신뢰 즉사. ❌ 영구 non-goal.

---

## 2. Value Proposition

### 판별러 (플레이어) — *"나, 몇 점짜리 눈인가?"*
- 매일 10문항으로 자기 감각을 **점수화**. 6/10이라는 자기 발견 자체가 상품.
- 문항마다 해설(tell) → 소모성 오락이 아니라 **판별 근육 훈련**. 연인 카톡,
  지원서, 리뷰, 보고서를 볼 때 실제로 쓰이는 감각.
- 🟩🟥 그리드 공유 = "내 눈 8/10, 너는?" — Wordle 문법의 경쟁 훅.

### 출제자 (작성자) — *"내 글은 아직 사람으로 읽히는가?"*
- 내 글이 같은 주제의 AI 답 옆에 놓였을 때 사람들이 구분해내면 →
  **인간성의 증명**. 못 구분해도 그것대로 이야깃거리.
- 같은 주제 비교 페이지(`/play/topic/*`): AI뿐 아니라 **다른 사람들 사이에서**
  내 글이 어떻게 읽히는지까지.
- 콘텐츠 공급이 곧 유저 보상 → 콘텐츠 CAC 0의 공급 루프.

### 왜 우리만 정직하게 줄 수 있나
탐지기(GPTZero류)는 확률 추측이라 필연적으로 오판하고, 오판은 무고한 사람을
낙인찍는다. 우리는 **출제 시점에 정답이 확정**되는 구조라 틀릴 수 없다.
"이 오라클은 구슬려지지 않는다"의 소비자판.

## 3. ICP (Ideal Customer Profile)

**공통 베이스:** 한국 20–35세 모바일 헤비유저. Wordle/꼬맨틀류 데일리 게임
문법에 익숙, 카톡·인스타 스토리·X 공유 문화, AI에 대한 호기심+불안 동시 보유.

| | ICP-1 판별러 (primary) | ICP-2 출제자 (supply) | ICP-3 불안 보유자 (growth) |
|---|---|---|---|
| 누구 | "나 AI 냄새 잘 맡음" 자부하는 개발자·마케터·에디터·대학생 | 글 좀 쓴다는 블로거·문예 취향·커뮤니티 헤비 라이터 | 과제 받는 교사/교수, 보고서 받는 팀장, 연애 중인 사람 |
| 동기 | 점수·랭킹·자기 증명 | "내 글은 AI와 다르다" 증명 욕구 | "그 메시지 진짜일까" 불안 해소 |
| 훅 | 리더보드 + 그리드 공유 | 결과 화면의 "직접 문제 내기" | "그 불안, 여기서 훈련하세요" 카피 |
| 전환 | 익명 플레이 (마찰 0) | 로그인 (서약·표기 필요) | 플레이 → ICP-1로 흡수 |

**Anti-ICP:** "이 실제 메시지 AI인지 판정해줘"를 원하는 사람. 의도적으로
서비스하지 않고 훈련 프레임으로 유도한다 (§1 제1원칙).

## 4. Moat

솔직한 전제: **게임 포맷 자체는 하루면 카피된다.** 해자는 포맷이 아니라
아래 네 겹이고, 전부 시간과 함께 두꺼워진다.

1. **검증된 사람-AI 쌍 코퍼스.** 무붙여넣기 + AI-미사용 서약 + 검토 승인을
   통과한 "확정 사람 글"과, 같은 주제로 생성된 AI 답의 **쌍**. 이런
   human-verified paired corpus는 GPTZero도 AI 랩도 갖고 있지 않다. 쌓일수록
   게임 신선도이자, 나중엔 그 자체로 연구/eval 자산.
2. **인간 혼동 데이터.** 수천 명의 플레이 기록 = "어떤 AI 글이 사람을 속이고,
   어떤 사람 글이 AI로 오인되는가"의 지도. 장기적으로 "당신 모델은 사람을 N%
   속입니다" 벤치마크(모델별 리더보드 → AI 랩 스폰서)로 이어지는 유일 자산.
3. **공급 루프.** 플레이어→출제자 전환으로 콘텐츠가 유저에게서 나온다.
   카피캣은 포맷은 베껴도 출제자 커뮤니티와 그들의 글은 못 가져간다.
4. **신뢰 포지션.** "정답 아는 것만 판정" 원칙. 탐지기들이 오판 스캔들로
   불타는 동안 우리는 구조적으로 틀릴 수 없다 — 이 카테고리에서 신뢰는
   한 번 잃으면 못 되사는 자산이라, 먼저 쌓는 쪽이 이긴다.

## 5. PRD

### 5.1 북극성 & 가드레일 지표
- **북극성: Daily Plays** (완주 기준).
- 공유율: 완주자 중 결과 공유 ≥ 15% (K-factor의 분자).
- 리텐션: D1 ≥ 30%, 스트릭 3일+ 유저 비중 추적.
- 공급: 플레이어→출제 전환 ≥ 3%, 주간 승인 문항 ≥ 20.
- **난이도 가드레일: 일평균 정답률 55–75% 밴드 유지.** 너무 쉬우면 승부감이
  없고, 너무 어려우면 좌절 이탈. 밴드 이탈 시 출제/승인 기준으로 조정.
- 무결성: 검토 반려율, (후일) 서약 위반 적발 건수.

### 5.2 Shipped (v1 — 현재 main)
- 데일리 게임: 결정적 일일 세트(5/5 균형), 서버 전용 정답지, 하루 한 판,
  🟩🟥 공유 그리드, 리더보드, 해설.
- 공급 루프: 출제(로그인) → AI 짝 생성(뱅크→키 넣으면 라이브 Claude) →
  검토 큐(`ADMIN_HANDLE`) → 승인 시 게임 풀 편입 + 공개 후 출처 표기.
- 무결성 장치: 붙여넣기/드롭 차단, AI-미사용·사용동의 이중 서약, 승인 게이트.
- 같은 주제 비교(`/play/topic/[id]`), Turso 영속 + `/api/health`.

### 5.3 Next (v1.1 — 이번 스프린트)
1. **홈 히어로 리테마** (이 PR): 첫 화면 = 판별 게임. escrow 마켓은 하단 유지.
2. 스트릭 + 개인 정확도 프로필("당신은 연애 카톡에 약해요") — 리텐션 코어.
3. 공유 카드 강화: 날짜별 OG 결과 카드(점수 그리드 이미지).
4. 카테고리별 통계 노출(오늘 평균, 가장 많이 속은 문항).

### 5.4 Later (v2+)
- 이미지 모달리티("이 사진 진짜야?") — 텍스트 루프의 공유율 검증 후.
- 모델별 리더보드: "어느 모델이 사람을 제일 잘 속이나" → AI 랩 스폰서 앵글.
- 서약 위반 이상탐지(통계적), 커뮤니티 신고.
- B2B: 인간 혼동 벤치마크 API (먼 미래, 데이터 쌓인 뒤).

### 5.5 Non-goals
- **실제 제3자 메시지 판정기.** (§1 제1원칙 — 영구.)
- 돈/베팅/에스크로 레이어 재도입 (트래픽 검증 전까지).
- 다국어 확장 (한국어 루프 검증 전까지).

### 5.6 Risks
| 리스크 | 대응 |
|---|---|
| 콘텐츠 반복/고갈 | 공급 루프가 본질 대응. 승인 문항 주간 목표로 모니터. |
| 서약 뚫고 AI 글 유입 (정답지 오염) | 붙여넣기 차단 + 서약 + 검토 큐. 후일 이상탐지. 오염 발견 시 해당 문항 즉시 회수. |
| "판정기" 기대 오용 | 모든 카피에서 "훈련 게임" 프레임 고정. 판정기 요청은 훈련으로 유도. |
| 포맷 카피캣 | 해자(§4)는 데이터·공급루프·신뢰 — 선점 속도가 방어. |

## 6. 홈 히어로 카피 (확정본)

```
eyebrow:  사람 vs AI · 매일 10문항 · 하루 한 판
H1:       당신의 눈,
          아직 통합니까?
sub:      연인의 카톡, 지원서, 별점 리뷰 — 이제 뭐든 AI가 썼을 수 있어요.
          매일 글 10개로 확인하세요. 탐지기의 추측이 아니라,
          출제 시점에 확정된 정답지로 채점합니다.
CTA-1:    오늘의 판별 시작 →   (/play)
CTA-2:    직접 문제 내기        (/play/submit)
proof:    오늘 {N}명 참여 · 평균 {X}/10 — 생각보다 어렵습니다
```

카피 원칙: ① 불안(연인·지원서·리뷰)을 훅으로 쓰되 약속은 **훈련**으로 한정
② "정답지" 차별점을 서브카피에 항상 명시 ③ 사회적 증거는 실데이터만
(가짜 숫자 금지 — 참여 0명이면 문구 자체를 숨긴다).
